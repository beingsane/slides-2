## Git, gitflow e GitHub ![logo git](images/logo_git.png) Turicas aka Álvaro Justen
# Por quê? - Manter **histórico** de desenvolvimento - Facilitar **colaboração** - Controle de **diferentes versões** - Adicionar **qualidade** ao processo de desenvolvimento
# VCS Centralizados - CVS - Subversion (aka SVN) - Clearcase (privativo) - Perforce (privativo)
# VCS Distribuídos - Git - Mercurial - Bazaar - BitKeeper (privativo)
## Centralizados ![Centralizados](images/flow-CVCS.png)
## Distribuídos ![Centralizados](images/flow-DVCS.png)
# Centralizados - Um **repositório central** - Um **diretório de trabalho** por colaborador - Commits **necessitam de rede**
# Distribuídos - Um **clone** por colaborador (repo + working dir) - Commits **locais** - **Sincronização** entre repositórios (push/pull)
Git, the information manager from hell.
Linus Torvalds

Git:

  • Criado por Linus Torvalds em 7 abril de 2005
  • Mantido por Junio Harmano aka gitster (Google) desde 26/07/2005
  • Distribuído (e desconectado)
  • Multiplataforma
  • Rápido, eficiente e poderoso
  • Software livre (GPL 2)
  • Facilita merges
  • Baseado em conteúdo, não em arquivos

Git: curiosidades

  • Repositório do git em git em 4 dias
  • Em 16 de abril de 2005 já estava versionando o Linux-2.6.12-rc2
