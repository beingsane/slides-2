## Aleph: Sugestões Turicas aka Álvaro Justen
## API não deve depender do FS - Obrigatoriedade de coletores na mesma máquina - Possíveis problemas de sincronização - Possíveis problemas de permissão - Impede escalabilidade
## Web-UI deve estar desacoplado - Disputa de CPU/memória - Possíveis problemas de permissão
## ElasticSearch pode ser um canhão - Mais uma parte móvel - Configuração mais complicada comparada com outras soluções
## Plugins não possuem teste - Medo de alterar o código e quebrar algo - Facilidade de bugs passarem despercebidos
## `settings.py` - Deveriam ser separados: um para `aleph-core`, outro para `webui` - Deveriam pegar dados de variáveis de ambiente (em vez de hard-coded)
## Deployment automatizado - [fabric](http://fabfile.org/)? - [docker](http://docker.io)?
