## Software Livre e Open Source são movimentos **muito diferentes**
## Copyleft - Força o software a **continuar livre** - Se alguém quer modificar o software e distribuí-lo, que a modificação **também seja software livre**
## Licenças - **Com copyleft**: GPL, CC-(SA)-*, ... - **Sem copyleft**: MIT, BSD, ...
## Problemas do não copyleft - Apple (darwin) - Microsoft Visual Studio Code (software privativo, spyware)
## Definição de Software Livre - 0: Uso para qualquer finalidade - 1: Modificação do software (requer código-fonte) - 2: Distribuição do software - 3: Distribuição do software modificado
## Definição de Open Source - Quase a mesma coisa (em mais itens)
## Qual a diferença, então?
## Objetivo dos movimentos - Software Livre: **liberdade do usuário**, do ecossistema - Open Source: **qualidade do software** - Desenvolver colaborativamente é uma forma de manter qualidade, mas o foco **não** é a liberdade do usuário
## Não saber a diferença pode fazer com que você esteja dedicando seu tempo a algo que você **não queira apoiar**
## Como melhoro isso? - Entendendo as **diferenças** e **objetivos** de cada movimento - Usando licenças com copyleft, caso você **se importe com a liberdade**
