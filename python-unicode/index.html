Ian Albert: impressão da tabela Unicode

Unicode [2]

  • Para "materializar": usar algum encoding!
  • Recomendado: UTF-8 (Unicode Transformation Format, 8-bit variable length)

Como resolver isso?

  • Regras de ouro:
    • Decoding o mais rápido possível
    • Trabalhe sempre com a representação em Unicode
    • Encoding o mais tarde possível

.encode/.decode

  • .encode: unicode → str
  • .decode: str → unicode

Entendendo o que está codificado

Python 2!

>>> print len('álvaro'), type('álvaro')
>>> print 'álvaro'[0]
>>> print 'álvaro'[1]
>>> print len(u'álvaro'), type(u'álvaro')
>>> print 'álvaro'.decode('utf-8') == u'álvaro'
>>> print 'álvaro'.upper(), type('álvaro'.upper())
>>> print 'álvaro'.decode('utf-8').upper(), \
          type('álvaro'.decode('utf8').upper())

ASCII

  • 128 símbolos
  • 000-032: caracteres de controle
  • 048-057: 0-9
  • 065-090: A-Z
  • 097-122: a-z
  • Não existe: áàãâä ç ć ...

>>> for numero in range(129):
...    print numero, chr(numero), ord(chr(numero))

UTF-8

  • Variable-length (começa com 8 bits)
  • Extensão do ASCII
  • Representa toda a tabela Unicode
  • Não possui BOM (Byte-Order-Marker, byte-endianness detection)
  • (UTF-16 possui)
  • (UTF-32 é mais rápido, mas ocupa mais espaço)

Resolvendo alguns problemas específicos

  • Cuidado com o fallback da fonte (lembra da Laís?)
  • Use unicodedata.normalize para fazer slugs
  • Cuidado: Nem todos encodings são baseados em ASCII!

Fazendo slugs da forma correta

>>> from unicodedata import normalize
>>> new_string = normalize('NFKD', u'Álvaro')
>>> print new_string.encode('ascii', 'ignore')
Alvaro

base64: encodando encodados

>>> # "ASCII armor"
>>> print u'Álvaro'.encode('utf-8').encode('base64')
w4FsdmFybw==
>>> 'w4FsdmFybw=='.decode('base64').decode('utf8')
u'\xc1lvaro'

Mais algumas dicas

  • Use as regras de ouro!
  • Faça testes de codificação
  • -, _ são ignorados no .encode/.decode
  • Use o header # coding: ...
  • Verifique se você pode representar o símbolo que você precisa na codificação desejada
  • Verifique qual codificação você recebeu
  • Explicite qual o encoding você está usando
  • Frameworks/bibliotecas deveriam fazer o trabalho sujo
